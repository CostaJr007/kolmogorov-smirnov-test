<available_tools>

**Available Tools:**
- **analyze_response (New):** Calculates Shannon Entropy and Information Gain between a baseline response and a target response.
  - **Parameters:**
    - `baseline_body`: The body of the normal, valid response.
    - `target_body`: The body of the response to your injection payload.
  - **Returns:** `{ entropy_baseline, entropy_target, entropy_diff, information_gain, length_diff, is_anomaly }`
- **save_deliverable (MCP Tool):** Saves deliverable files with automatic validation.
  - **Parameters:**
    - `deliverable_type`: "XSS_ANALYSIS" or "XSS_QUEUE" (required)
    - `content`: Your markdown report or JSON queue (required)
  - **Returns:** `{ status: "success", filepath: "...", validated: true/false }` on success or `{ status: "error", message: "...", errorType: "...", retryable: true/false }` on failure
  - **Usage:** Call the tool with your deliverable type and content. Queue files must have `{"vulnerabilities": [...]}` structure and will be validated automatically.
- **{{MCP_SERVER}} (Playwright):** To interact with the live web application, submit forms, and capture responses.
- **TodoWrite Tool:** Use this to create and manage your analysis task list.

<methodology>

## **Gray-box XSS Analysis (Information Theory Enhanced)**

- **Goal:** Identify XSS vulnerabilities by detecting reflections and anomalies in server responses.
- **1) Create a To Do for each Injection Source found in the Recon Deliverable**
- **2) Establish Baseline:**
    - Send a valid, benign request to the target endpoint.
    - Capture the response body. This is your `baseline_body`.
- **3) Fuzzing & Differential Analysis:**
    - Send requests with XSS polyglots and probe strings (e.g., `"><img src=x onerror=alert(1)>`).
    - Capture the `target_body`.
    - Use `analyze_response(baseline_body, target_body)` to get anomaly metrics.
- **4) Interpret Results:**
    - **Reflection:** Check if your payload (or parts of it) appears in `target_body`.
    - **High Information Gain (> 0.8):** Significant structural change. Could be a WAF block, error page, or successful injection.
    - **Context Analysis:** If reflected, where? (Inside a tag? Attribute? Script?).
- **5) Make the Call:**
    - **Vulnerable:** Payload reflected without proper encoding for the context, OR significant anomaly suggesting blind XSS/WAF interaction.
    - **Safe:** Payload fully encoded or not reflected; response matches baseline structure.
- **6) Document Finding:**
    - Use `exploitation_queue_format`.
</methodology>
